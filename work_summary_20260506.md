## 1. 주요 개발 및 개선 사항

### 🚀 슈퍼 어드민(Super Admin) 대시보드 완성
- **목적**: 시스템 전체를 관리할 수 있는 강력한 관리 도구 구축
- **핵심 기능**:
  - **대시보드**: 총 사용자, 사업자, 수업 수 및 최근 로그인 활동 실시간 모니터링
  - **사용자 관리**: 이름/이메일 검색, 권한 필터링, **사용자 권한(USER/BUSINESS/ADMIN) 변경**, 계정 삭제 기능
  - **수업 관리**: 전체 업체에서 등록한 수업 목록 조회 및 불필요한 수업 강제 삭제
  - **시스템 로그**: 최근 100건의 상세 로그인 기록(방법, 플랫폼, 시간) 추적
  - **설정**: 시스템 버전 정보 및 데이터 현황 요약 제공

### 🔐 로그인 및 가입 프로세스 최적화
- **모바일 구글 로그인 안정화**: 모바일 환경에서 팝업 차단으로 로그인이 실패하는 문제를 해결하기 위해 `signInWithRedirect` 방식을 도입하고 상태 복구 로직을 추가했습니다.
- **구글 최초 가입 시 권한 선택**: 구글로 처음 로그인하는 신규 사용자가 자신의 역할을 '일반 사용자' 또는 '사업자' 중 직접 선택할 수 있도록 **미려한 디자인의 전용 모달**을 구현했습니다.
- **아이디 저장 기능 (Remember Me)**: 로컬 스토리지를 활용하여 다음에 접속할 때 이메일 주소를 자동으로 입력해주는 편의 기능을 추가했습니다.

## 2. 발생 문제 및 원인 분석 (RCA)

| 문제 현황 | 원인 분석 (Root Cause) | 대책 및 해결 방법 |
| :--- | :--- | :--- |
| **Vercel 구글 로그인 실패** (`unauthorized-domain`) | Firebase 보안 설정에 Vercel 배포 도메인이 '승인된 도메인'으로 등록되지 않음 | Firebase 콘솔의 Authentication 설정에서 Vercel 주소 등록 안내 |
| **새로고침 시 404 에러** | SPA(Single Page App) 특성상 서버에 해당 경로의 실제 파일이 없어 발생하는 라우팅 이슈 | 프로젝트 루트에 `vercel.json` 파일을 생성하여 모든 요청을 `index.html`로 리다이렉트 처리 |
| **구글 가입 시 권한 고정** | 신규 사용자가 로그인 탭에서 구글 로그인을 누르면 무조건 'USER'로 가입되도록 하드코딩됨 | `isLogin` 상태와 관계없이 신규 유저라면 무조건 역할 선택 모달을 띄우도록 로직 변경 |
| **JSX 네임스페이스 에러** | React 19와 TypeScript 설정 간의 버전 호환성 및 타입 정의 누락 | `React` 명시적 임포트 및 `React.ReactNode` 타입 사용으로 정적 분석 오류 해결 |
| **모바일 팝업 차단** | 브라우저 보안 정책상 사용자 인터랙션 없는 팝업 창 차단 | 기기 감지 로직을 추가하여 모바일 접속 시에는 리다이렉트 방식으로 자동 전환 |

## 3. 기술적 대책 및 유지보수 가이드

### 🛠️ 배포 및 인프라
- **vercel.json 관리**: 새로운 경로가 추가되더라도 SPA 라우팅이 유지되도록 `vercel.json` 파일의 `rewrites` 설정을 항상 유지해야 합니다.
- **Firebase 도메인 관리**: 테스트 도메인이나 정식 도메인이 바뀔 때마다 Firebase Console에서 해당 도메인을 반드시 승인 목록에 추가해야 인증 기능이 정상 작동합니다.

### 💻 코드 품질 유지
- **미사용 변수 제거**: 이번 작업에서 `mobileMenuOpen`, `isLoginMode` 등 불필요해진 변수들을 대거 정리했습니다. 향후 개발 시에도 `unused-vars` 경고를 상시 체크하여 코드 청결도를 유지합니다.
- **반응형 설계**: 모든 대시보드(Super Admin, Business, User)는 Tailwind CSS를 활용해 모바일과 데스크톱을 동시에 지원하도록 설계되었습니다. 새로운 UI 추가 시에도 이 기조를 유지합니다.
